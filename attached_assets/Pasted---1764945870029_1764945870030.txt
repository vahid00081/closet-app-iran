گام اول: انتخاب و راه‌اندازی ابزار
برای موفقیت در وایب کدینگ، نیاز به یک ابزار با قابلیت‌های بالا دارید که بتواند کل پروژه‌ی شما را درک و ویرایش کند:

نصب یا استفاده از ابزار: توصیه می‌کنم از ابزاری مانند Cursor (برای بهترین تجربه، اگرچه ابزارهای تحت وب مانند Replit یا Bolt.new هم کار می‌کنند) استفاده کنید.

ایجاد یک پروژه جدید: یک پوشه (Folder) خالی ایجاد کنید و آن را با Cursor باز کنید، یا یک پروژه جدید React در Replit ایجاد کنید.

انتخاب مدل هوش مصنوعی: مطمئن شوید که از یک مدل قوی مانند Claude 3.5 Sonnet یا GPT-4o استفاده می‌کنید.

گام دوم: پرامپت اولیه (نقشه راه برای هوش مصنوعی)
مهم‌ترین بخش وایب کدینگ، نوشتن یک پرامپت کامل است. این پرامپت "وایب" و تمام نیازمندی‌های تکنیکال و عملکردی شما را مشخص می‌کند.

این متن را کپی کنید و در قسمت چت هوش مصنوعی (Composer در Cursor) وارد کنید:

پرامپت اصلی برای ساختاردهی پروژه

"من می‌خواهم یک وب‌اپلیکیشن فول‌استک (Full-stack) با تمرکز بر موبایل (Mobile-First) با استفاده از React و Tailwind CSS توسعه دهم. هدف این اپلیکیشن، تبدیل شدن به یک دستیار هوشمند لباس است.

بخش A: هسته فنی و معماری

ساختار: یک ساختار تمیز برای یک پروژه Next.js یا React + Vite با TypeScript ایجاد کن.

استایل: تمام استایل‌دهی باید با Tailwind CSS انجام شود.

ذخیره‌سازی (Mock Storage): برای مدیریت داده‌های کمد لباس، یک Service یا Hook بساز که از localStorage یا IndexedDB مرورگر استفاده کند تا داده‌ها در سیستم کاربر ذخیره شوند.

بخش B: پیاده‌سازی ویژگی‌ها

سرویس آب و هوای ساختگی (Mock Weather Service):

به جای اتصال به API واقعی، یک کامپوننت و سرویس ساختگی بنویس که دمای فعلی را همیشه روی ۱۸ درجه سانتی‌گراد نمایش دهد.

این سرویس باید قابلیت تغییر دستی به سه حالت (وایب) زیر را داشته باشد: سرد (Cold: ۵°C)، معتدل (Mild: ۱۸°C)، گرم (Warm: ۳۰°C).

ماژول کمد لباس (The Closet):

آپلود: یک صفحه برای آپلود عکس لباس ایجاد کن.

تگ‌گذاری: کاربر باید بتواند برای هر لباس، تگ‌های زیر را با استفاده از Dropdown یا Checkbox انتخاب و ذخیره کند:

نوع: (بالاتنه، پایین‌تنه، کفش، اکسسوری)

مناسب دما: (سرد، معتدل، گرم، همه)

منطق پیشنهادی (The Recommendation Engine):

یک تابع بنویس که بر اساس دمای فعلی (از سرویس Mock)، حداقل یک آیتم از هر دسته‌بندی (بالاتنه، پایین‌تنه، کفش) را که با آن دما تگ شده، به صورت تصادفی انتخاب کند.

بخش C: وایب و طراحی (The Vibe)

طراحی باید مینیمال و تمیز باشد. از Dark Mode به عنوان پیش‌فرض استفاده شود.

گوشه‌های دکمه‌ها و کارت‌ها باید گرد (Rounded) باشند.

فایل‌های مورد نیاز: لطفاً ساختار پروژه، تنظیمات Tailwind و فایل‌های اصلی برای صفحات Dashboard، Closet و سرویس‌های Storage و MockWeather را ایجاد کن."

گام سوم: اجرای پرامپت و تایید تغییرات
Execute (اجرا): دکمه اجرای پرامپت (مثلاً Generate یا Apply در Cursor) را بزنید.

Review (بررسی): هوش مصنوعی شروع به ساخت تمام فایل‌های پروژه (package.json, tailwind.config.js, کامپوننت‌های React و غیره) می‌کند.

Apply (اعمال): اگر از تغییرات راضی بودید، دکمه اعمال تغییرات را بزنید تا کدها در پوشه شما ذخیره شوند.

گام چهارم: اجرا و رفع ایرادات (Iteration)
حالا پروژه شما آماده است. آن را اجرا کنید:

نصب وابستگی‌ها: دستوراتی که هوش مصنوعی برای نصب داده (مثل npm install یا yarn) را در ترمینال اجرا کنید.

اجرای پروژه: با دستور npm run dev یا معادل آن، اپلیکیشن را اجرا کنید.

بازبینی و اصلاح (Refinement):

اگر در اجرا خطایی دیدید، متن خطا را کپی کرده و به هوش مصنوعی بگویید: "لطفاً این خطا را در فایل [نام فایل] برطرف کن."

اگر ظاهر به اندازه کافی مینیمال نیست، بگویید: "رنگ دکمه آپلود خیلی تیره است، آن را به یک رنگ سبز پاستلی ملایم با یک سایه کوچک تغییر بده."

تست منطق: یک لباس با تگ "سرد" آپلود کنید و حالت آب و هوا را روی "گرم" بگذارید. اگر لباس را پیشنهاد داد، به هوش مصنوعی بگویید: "منطق پیشنهادی خطا دارد، در دمای گرم نباید لباس‌هایی با تگ سرد نمایش داده شوند."

گام نهایی: اتصال به API واقعی
هنگامی که تمام منطق پیشنهادی با موفقیت روی دیتای ساختگی (Mock Data) کار کرد، می‌توانید مرحله نهایی را انجام دهید:

دریافت کلید API: یک کلید رایگان از OpenWeatherMap بگیرید.

دستور اتصال: به هوش مصنوعی بگویید: "حالا که همه چیز کار می‌کند، لطفاً سرویس Mock Weather را با اتصال به OpenWeatherMap API جایگزین کن. منبع دمای واقعی باید جایگزین دمای ساختگی شود. این API Key من است: [کد API خود را اینجا وارد کنید]."
